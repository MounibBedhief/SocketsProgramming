# 🌐 Client/Server Programming with Sockets (C)

## 🧾 Overview
This project was carried out as part of a practical assignment in client/server programming. Its main objective is to explore and implement network communications using the socket interface in C, through both connection-oriented (TCP) and connectionless (UDP) models.

The work focuses on understanding how distributed applications communicate over a network, from low-level data exchange to higher-level protocols such as HTTP.

---

## 📚 Context
Modern networked applications rely on well-defined communication models to ensure data exchange between distributed systems. This project provides a hands-on approach to these concepts by implementing and analyzing client/server architectures.

Particular attention is given to:
- the behavior of TCP and UDP protocols 🔁  
- the structure and exchange of HTTP messages 🌍  
- the lifecycle of a network connection 🔗  
- the challenges of handling multiple clients concurrently ⚙️  

Network traffic analysis tools are also used to observe real communications at different layers (application, transport, and network).

---

## 🛠️ Scope of Work
The project includes several key components:

- Development of a basic HTTP client capable of sending requests and displaying server responses 📡  
- Implementation of a TCP-based client/server application for reliable message exchange 🔒  
- Implementation of a UDP-based version to highlight differences in communication behavior ⚡  
- Design of a concurrent server capable of handling multiple clients simultaneously 🧵  

---

## 🎯 Key Learning Outcomes
Through this project, the following concepts are explored and applied:

- Socket programming in C 💻  
- TCP connection establishment and termination 🔄  
- UDP datagram-based communication 📦  
- HTTP request/response structure 📑  
- Network traffic analysis and interpretation 🔍  
- Concurrent server design 🧠  

---

## ✅ Conclusion
This project provides a practical foundation in network programming by bridging theoretical concepts with real-world implementation. It highlights the fundamental differences between communication protocols and introduces key design considerations for scalable client/server systems.
